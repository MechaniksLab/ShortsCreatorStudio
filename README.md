# Лаборатория Механика — Студия создания шортсов

> Мощная студия для создания Shorts/Reels/TikTok из длинных видео: AI-автонарезка, удаление неречевых участков, стильные субтитры, пакетная обработка и гибкая настройка качества.

## 🚀 Что это за проект

**ShortsCreatorStudio** (в интерфейсе: **«Лаборатория Механика — Студия создания шортсов»**) — это развитие идеи [VideoCaptioner](https://github.com/WEIFENG2333/VideoCaptioner), ориентированное на быстрый выпуск short-form контента.

Проект подходит, если вам нужно:

- быстро разрезать длинное видео на потенциально «цепляющие» моменты;
- автоматически обработать речь и подготовить субтитры;
- визуально настроить монтаж под вертикальный формат;
- массово прогонять пачки файлов в полуавтоматическом режиме.

## ✨ Ключевые возможности

### 1) Создание шортсов (Auto Shorts)

- автоматический поиск интересных фрагментов по распознанной речи;
- ручной отбор найденных кандидатов перед финальным рендером;
- анти-дубли: фильтрация похожих моментов;
- контроль границ фраз (чтобы не резало слова);
- ограничение диапазона анализа по таймлайну;
- выбор настроек рендера (backend/FPS/разрешение/качество);
- предпросмотр и сохранение шаблона раскладки (между запусками).

### 2) Визуальный шаблон монтажа

- интерактивные зоны **WEBCAM/GAME**:
  - кроп исходного кадра;
  - позиционирование в вертикальном холсте 1080×1920;
- realtime предпросмотр компоновки;
- цветокоррекция слоёв:
  - яркость,
  - контраст,
  - насыщенность,
  - резкость;
- сохранение/загрузка шаблона.

### 3) Создание и стилизация субтитров

- транскрибация речи в текст;
- оптимизация текста и перевод (в зависимости от выбранного сервиса);
- расширенные стили субтитров (включая эффекты и читаемость);
- подготовка пресетов под разные типы контента;
- работа с шаблонами для единообразия роликов.

### 4) Пакетная обработка

Поддерживаются режимы:

- **Транскрибация** — только распознавание речи в текст;
- **Субтитры** — генерация/применение субтитров для готового видео;
- **Full process** — полный цикл в одном проходе:
  1. распознавание речи,
  2. обработка текста (оптимизация/перевод по настройкам),
  3. формирование субтитров,
  4. рендер финального результата.

### 5) Переозвучка / перевод видео (Video Translate)

- каскад распознавания + перевод + синтез нового голоса;
- поддержка voice clone через **Coqui XTTS**;
- native-конвертация **RVC** (через отдельный runtime для совместимости);
- source separation: **Demucs** и **UVR/MDX** с fallback-режимами;
- fallback-логика по ключевым этапам (ASR/diarization/TTS), чтобы пайплайн не падал целиком.

### 6) Система Антимат

- анализ речи из выбранного видео/аудио через Whisper с таймингами слов;
- детекция нецензурной брани через комбинированный подход:
  - словарь запрещённых слов,
  - LLM-анализ текста,
  - принудительный список разрешённых слов (исключения);
- полуавтоматический монтажный режим: таблица найденных областей с ручным включением/исключением;
- автоматический режим: анализ + рендер без ручной правки;
- выбор устройства распознавания: **GPU (CUDA)** или **CPU**;
- галочка кэширования ASR (ускоряет повторные прогоны);
- обязательное уведомление пользователя при отклонении от выбранного LLM-режима (подтверждение перехода в режим только списков);
- подробные прогресс-статусы по этапам и подсказки (tooltips) для ключевых настроек;
- фильтры таймингов цензуры:
  - захват до слова (ms),
  - захват после слова (ms),
  - склейка соседних матных зон (ms),
  - минимальная и максимальная длительность одной зоны цензуры;
- режимы цензуры:
  - `beep` (запикивание),
  - `mute` (заглушение);
- расширенные фильтры запикивания:
  - профили `classic`, `dual`, `soft`, `radio`, `noise`,
  - регулировка частоты beep,
  - регулировка громкости beep,
  - регулировка уровня подмешивания исходной речи (ducking);
- предпрослушивание beep-профиля перед финальным рендером;
- смешанный рендер: часть областей можно запикивать, часть — глушить.
- интерфейс разделён по этапам применения фильтров:
  - блок фильтров этапа 1 + кнопка «Анализ речи и поиск мата»,
  - блок фильтров этапа 2 + кнопка «Применить цензуру»;
- списки слов перенесены в отдельные диалоги:
  - кнопка «Запрещённые слова»,
  - кнопка «Разрешённые слова»;
- предзаполнение словарей по умолчанию и сохранение пользовательских списков;
- поддержка пресетов Антимата (сохранение/загрузка всех фильтров и параметров).

## 🧭 Вкладки и что в них делать

### Главная → **Создание шортсов**

Используйте, когда нужно быстро нарезать ролик на короткие клипы. Подходит для больших исходников (стримов, подкастов, интервью, летсплеев).

### Главная → **Создание субтитров**

Стартовая точка для работы с одиночным файлом: импорт видео/аудио, запуск pipeline распознавания и дальнейшей обработки.

### Главная → **Распознавание речи**

Настройки и выполнение ASR-этапа: выбор модели, проверка качества распознавания, подготовка текста для последующих шагов.

### Главная → **Оптимизация и перевод субтитров**

Тонкая корректировка и перевод текста субтитров. Здесь настраивается «читаемость» и итоговая формулировка реплик.

### Главная → **Синтез видео с субтитрами**

Финальный рендер с наложением выбранного стиля субтитров и параметров видео.

### Главная → **Переозвучка/Перевод видео**

Модуль локальной переозвучки: перевод реплик, сборка voice reference, синтез, опциональный RVC-финиш и экспорт итоговой дорожки/видео.

### Главная → **Система Антимат**

Модуль автоматической/ручной цензуры нецензурной брани: анализ речи, разметка по таймингам и финальный рендер с beep/mute.

### Отдельный раздел → **Пакетная обработка**

Для массовых задач: быстро прогонять десятки файлов в одном режиме, без ручного повторения одинаковых действий.

### Отдельный раздел → **Стиль субтитров**

Центр управления внешним видом субтитров: эффекты, цвета, композиция, пресеты и предпросмотр.

### Отдельный раздел → **Настройки**

Глобальные параметры программы:

- темы (светлая/тёмная/системная);
- акцент и основные пользовательские цвета интерфейса;
- LLM/переводчики/ASR-конфиги;
- рабочие директории и системные параметры.

## 🧩 Актуальные модули интерфейса

`app/view/`:

- `main_window.py` — главное окно, навигация, проверка обновлений;
- `home_interface.py` — контейнер ключевых вкладок;
- `auto_shorts_interface.py` — создание шортсов;
- `task_creation_interface.py` — создание субтитров (старт пайплайна);
- `transcription_interface.py` — распознавание речи;
- `subtitle_interface.py` — оптимизация/перевод субтитров;
- `video_synthesis_interface.py` — синтез видео с субтитрами;
- `video_translate_interface.py` — переозвучка/перевод видео;
- `anti_mate_interface.py` — система антимата (поиск мата + цензура beep/mute);
- `batch_process_interface.py` — пакетная обработка;
- `subtitle_style_interface.py` — стиль и пресеты субтитров;
- `setting_interface.py` — настройки приложения;
- `log_window.py` — просмотр логов.

Ключевые backend-пакеты в `app/core/`: `shorts`, `anti_mate`, `subtitle_processor`, `video_translate`, `bk_asr`, `storage`, `utils`.

Дополнительно по архитектуре:

- `app/thread/` — фоновые worker-потоки (ASR, batch, synthesis, video translate, автошортсы, проверка версий и т.д.);
- `app/components/` — переиспользуемые UI-компоненты и карточки настроек;
- `app/common/` — общая конфигурация, сигналы, тема;
- `scripts/` — утилиты установки/диагностики окружений и служебные скрипты pipeline.

## 🧠 Как устроен рабочий pipeline

Типичный сценарий:

1. Импорт исходного файла.
2. Распознавание речи (ASR).
3. Поиск и отбор значимых фрагментов.
4. Анти-дубликаты и чистка неречевых участков.
5. Рендер шортсов / или генерация итогового видео с субтитрами.
6. (Опционально) Пакетный прогон для серии файлов.

## 📦 Инструкция по запуску (правильный способ)

Важно: в архиве релиза уже лежит **готовая сборка** — исполняемый файл, runtime-окружения, ресурсы и предзагруженные модели (включая модели для video translate/UVR, если они входят в конкретный релиз).

Репозиторий в первую очередь содержит исходный код и скрипты обслуживания. Для обычного пользователя правильный путь — запуск из **архива Releases**.

Поэтому запуск делается так:

1. Скачайте архив программы из раздела **Releases** (в нём уже есть всё необходимое для старта).
2. Распакуйте архив в отдельную папку.
3. Запустите `ShortsCreatorStudio.exe`.
4. Обновить программу, если она предложит.

### Важно для модуля «Создание шортсов» (LLM)

Для поиска лучших моментов в **«Создание шортсов»** нужно настроить LLM в **Настройки → Параметры LLM**:

- можно использовать удалённую модель (OpenAI/другие совместимые провайдеры);
- можно использовать локальную модель, например через **LM Studio**.

Пример с LM Studio:

1. Установите LM Studio.
2. Скачайте в LM Studio подходящую модель.
3. Запустите локальный сервер LM Studio (Local Server).
4. В программе выберите сервис **LM Studio** в настройках LLM.
5. Укажите Base URL LM Studio (обычно `http://localhost:1234/v1`).
6. Выберите модель, которая будет использоваться для отбора шортсов.

Без настроенной LLM-модели качество/работа автоматического отбора моментов для шортсов будет ограничена.

### Важно для модуля «Перевод видео» (voice clone)
В проект добавлен production-подход для установки зависимостей video translate под разные ПК:

- `scripts\\install_video_translate_full_auto.cmd` — **полный авто-режим**: определяет тип GPU, ставит основной стек (GPU-auto или CPU), затем пытается подготовить отдельный `runtime_rvc` для native RVC;
- `scripts\\install_video_translate_cpu.cmd` — стабильный CPU-профиль (работает почти везде);
- `scripts\\install_video_translate_gpu_auto.cmd` — авто-выбор GPU профиля (пытается cu128 → cu124 → fallback на CPU);
- `scripts\\install_video_translate_gpu_cu124.cmd` — принудительный GPU-профиль CUDA 12.4;
- `scripts\\install_video_translate_gpu_cu128.cmd` — принудительный GPU-профиль CUDA 12.8;
- `scripts\\install_video_translate_deps_only.cmd` — установка/обновление core-пакетов (XTTS/faster-whisper/pyannote/demucs) без переустановки torch;
- `scripts\\install_video_translate_studio_addons.cmd` — безопасные studio-дополнения для улучшения микса/шумоподавления (pyloudnorm, noisereduce, pedalboard, audio-separator);
- `scripts\\install_video_translate_experimental_addons.cmd` — экспериментальные дополнения (WhisperX/Resemblyzer/WebRTCVAD) в best-effort режиме + авто-восстановление production core-версий;
- `scripts\\check_video_translate_env.py` — диагностика готовности окружения.

По runtime для video translate:

- основной runtime: `runtime` (сейчас обычно Python 3.11) — XTTS/ASR/diarization/общий стек;
- RVC runtime: `runtime_rvc` (обычно Python 3.10) — отдельный runtime для совместимости `rvc-python/fairseq`;
- в интерфейсе можно явно задать путь `VideoTranslate -> RVCRuntimePython`.

Это нормальный сценарий для текущей версии, потому что часть RVC-стека нестабильна на Python 3.11.

Как выбрать установку под вашу видеокарту:

- **NVIDIA (не уверены в версии CUDA)** → запускайте `install_video_translate_full_auto.cmd` (рекомендуется);
- **NVIDIA (точно знаете CUDA 12.8 / 12.4)** → можно использовать `...gpu_cu128.cmd` / `...gpu_cu124.cmd`;
- **AMD / Intel / встроенная графика / сервер без GPU** → используйте `install_video_translate_cpu.cmd`;
- если нужен только апдейт моделей/пакетов без смены torch-профиля — `install_video_translate_deps_only.cmd`.

Рекомендуемый порядок:

1. Базово для большинства пользователей: `scripts\\install_video_translate_full_auto.cmd`.
2. Если хотите ручной контроль профиля torch: `...gpu_auto.cmd` / `...gpu_cu124.cmd` / `...gpu_cu128.cmd` / `...cpu.cmd`.
3. Для обновления только voice/ASR/diarization/separation-стека — `scripts\\install_video_translate_deps_only.cmd`.
4. Для «студийного» качества микса/очистки — `scripts\\install_video_translate_studio_addons.cmd`.
5. Экспериментально (опционально): `scripts\\install_video_translate_experimental_addons.cmd`.
6. Проверяйте состояние через `scripts\\check_video_translate_env.py`.

Важно по experimental-пакетам:

- `WhisperX` может тянуть другие версии `faster-whisper/pyannote` и потенциально конфликтовать с production-пином;
- `Resemblyzer` иногда подтягивает `webrtcvad` со сборкой C-extension на Windows;
- поэтому experimental-установка сделана как best-effort (ошибки отдельных пакетов не блокируют весь сценарий);
- после experimental-установки скрипт автоматически возвращает production-пины (`faster-whisper==1.2.1`, `pyannote.audio==4.0.4`), чтобы не деградировал основной пайплайн.

Технически модуль поддерживает fallback-режимы:

- ASR: `faster-whisper` CLI или python fallback;
- speaker diarization: `pyannote` (high) или heuristic fallback;
- voice clone: Coqui XTTS (high) или базовый fallback-режим.

Дополнительно для максимально похожего дубляжа (новый каскад quality-mode):

- source separation: **Demucs + UVR/MDX (Inst_HQ_3 + Kim_Vocal_2)** с авто-fallback;
- улучшенная сборка voice reference: фильтрация тихих/плохих сегментов + мягкая очистка перед XTTS;
- выбор режима separation в UI модуля «Перевод видео»: Demucs+UVR / Demucs / UVR / Auto;
- UVR модели автоматически подтягиваются в `AppData/models/uvr` при запуске install `*.cmd` для video translate.

Это и есть рекомендованный сценарий обновления/запуска на текущий момент.
